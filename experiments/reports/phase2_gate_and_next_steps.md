# Phase 2 ゲート判定と今後の方針

作成: 2026-07-12 / 対象 Issue: #27(cpt-B)・#75(層プロファイリング)・Epic #4

## 1. 背景

設計書(docs/lfm2_5-ja-plan.md)の Phase 2 は「LFM2.5-1.2B に日本語 CPT
(継続事前学習)を施すことが日本語性能の向上に寄与するか」を 3 実験の比較で判定する:

- **cpt-A**: CPT なし対照(1.2B-Base をそのまま評価)
- **cpt-B**: 1.2B-Base + 日本語 CPT(wiki ja+en ~318M tok、選択層フルパラメータ学習)
- **cpt-C**: 公式日本語版(LFM2.5-1.2B-JP-202606)を起点に SFT のみ(CPT スキップ)

ゲートルール(設計書): cpt-B が cpt-C を上回らなければ、以降のフェーズは cpt-C 起点に切り替える。

あわせて、論文 arXiv:2607.01232「Is One Layer Enough?」の主張
(post-training の効果はモデル中央の少数層に集中し、両端の寄与は小さい)が
**ハイブリッド構造(conv 10 層 + attention 6 層)の LFM2.5 でも成立するか**を、
全 16 層の単層学習スイープで検証した(#75)。これは cpt-B の可変層 [7,8] の妥当性確認であると
同時に、Phase 3 以降の層選択の根拠づくりでもある。

## 2. 判定材料(確定済みの実測値)

### 2.1 cpt-B 完走モデルの評価(Stage V、reports/phase2_cpt_b_final_eval.md)

| 指標 | cpt-A(Base) | cpt-B final(12,941 steps) | cpt-C(JP-202606) |
|---|---|---|---|
| held-out ja ppl | 9.12 | **8.36**(単調改善) | - |
| 日本知識プローブ /50 | 25 | 26 | - |
| llm-jp-eval AVG(凍結 8 タスク) | 0.235 | 0.216 | **0.387** |

- ppl とプローブは改善する一方、**llm-jp-eval は CPT 前より低下**(jsts 負相関化、
  jsick/jnli/niilc 低下。例外的に jmmlu は 0.05→0.27 に改善)
- 解釈: 生 wiki コーパスの CPT は言語モデリング能力(ppl)を上げるが、few-shot の
  構造化出力に従う能力を毀損する。知識注入の効果はプローブ +1 点相当と小さい

### 2.2 層プロファイリング(reports/phase2_layer_profiling.md)

- 論文の中央集中パターンが**ハイブリッドでも再現**: 改善するのは中央帯 L6〜L9 のみ、
  両端は悪化(L15 は train loss が下がらず適合不能)
- deci(32M tok)で **L9・L6(いずれも conv)が最良**、L7 次点、L8(attn)劣後。
  位置 > 層タイプ、かつ中央帯内では conv 優位
- プローブ採点は centi+deci 計 1,000 問を手動照合し、過大評価ゼロを確認済み

## 3. ゲート判定

**cpt-B(生 wiki CPT)ルートは不採用とし、設計書ルールどおり cpt-C
(LFM2.5-1.2B-JP-202606 起点)を Phase 3 以降の基盤に採用する。**

根拠:

1. 凍結評価(llm-jp-eval)で cpt-B 0.216 vs cpt-C 0.387 と**全面劣後**。差(0.17)は
   層選択の最適化で得られる改善幅(deci で Δppl 0.09 = プローブ数点相当)より一桁大きい
2. cpt-B の弱点(構造化出力の毀損)は SFT で一部回復し得るが、cpt-C は同じ SFT を
   より高い出発点から積める。cpt-B を選ぶ積極的理由がない
3. 層プロファイリングの結論は「どの層でも wiki CPT の質的限界は変わらない」ことを示した
   (全 20 モデルが同じ知識問題で誤答)。ボトルネックは層選択ではなくデータ配合にある

なお cpt-B の資産(packed cache 基盤、冪等スイープドライバ、quick_eval/japan_probe、
中断耐性の運用ノウハウ)はすべて Phase 3 以降でそのまま使う。

## 4. 今後の方針

### 4.1 Phase 3(SFT)— 次の主戦場

- **起点モデル**: LFM2.5-1.2B-JP-202606(cpt-C)
- **層選択**: 本プロファイリングの知見を転用し、sft-003(#35)の比較群を
  「単層 L9 / 単層 L6 / 2 層 [6,9] / 中央 4 層 [6..9] / フル FT 参照」で構成する。
  端の層・L15 は比較群から除外してよい(centi で適合不能を実証済み)
- 実装順: SFT 学習スクリプト(#31)→ 設定群(#32)→ sft-001 ichikara(#33)→ 以降スイープ
- 学習方式は引き続き「選択層のフルパラメータ学習」(LoRA 不使用)、seq 6144・WSL2

### 4.2 CPT 再挑戦の条件(オプション、優先度低)

生 wiki CPT の敗因はデータ配合にあるため、再挑戦するなら層選択ではなく配合を変える:
instruction 形式の混合(few-shot 耐性の維持)、llm-jp-corpus 等の多様なソース、
リプレイ(元分布データの混入)。ただし Phase 3 の SFT で日本語性能が十分伸びるなら不要。

### 4.3 直近の残タスク

1. **#66**: llm-jp-eval の形式不整合(jsem ool 100% / jmmlu JP ool 55%)の修正。
   Phase 3 のゲート評価を同じハーネスで行うため、SFT 開始前に解消しておくのが望ましい
2. Phase 2 の後片付け: 監視定期ジョブの無効化、スイープ中間生成物(WSL outputs/sweep*)の
   ディスク整理(モデル本体は削除可、RESULT.json/ログは保全)
3. Phase 4(DPO)・Phase 5(量子化/リリース)は設計書どおり Phase 3 の結果を待って着手
