# Phase 3 sft-004 lr/epoch スイープ

Issue #36。sft-003 層 ablation(#33 #35、PR #86〜#100、
`experiments/reports/phase3_sft_layer_ablation.md`)で 5 アーム全てが
pre-SFT base(`LiquidAI/LFM2.5-1.2B-JP-202606`、AVG 0.469)を下回った件を
受け、「学習強度が可変 params 数に対して過剰」仮説を直接テストする。
データ・層構成を固定し、lr × epoch のみを振る。sft-002(#34、データ多様化)
より先に実施(1 変数ずつ動かす切り分けを最優先)。

## 背景要旨

sft-003 の下流評価(llm-jp-eval 凍結条件・8 タスク × 100 件 × 4-shot × greedy)は
以下の順位逆転を示した:

| 順位 | アーム | train loss (epoch2 平均) | llm-jp-eval AVG | Δ vs base |
|---|---|---:|---:|---:|
| loss最良 | [6..9] | 0.990 | 0.324 | -0.145 |
| | [6, 9] | 1.103 | 0.403 | -0.066 |
| | full | 1.197 | 0.089 | -0.380 |
| | L9 | 1.276 | 0.396 | -0.073 |
| loss最悪 | L6 | 1.383 | 0.410 | -0.059 |

**train loss と AVG がほぼ完全に逆転**し、L6(最も学習されなかった)が首位。
生成テキスト 288 件の目視検査(検証A)で機序が確定: 劣化は可変 params 数に
比例した 3 段階のグラデーション(様式ドリフト → エコー/反復ループ → 事実崩壊)。
文法・流暢性は無事。chat_template 不一致仮説は棄却済み(PR #88、generated diff=0)。

### 動作点判定

**判定基準**: AVG ≥ 0.469(base 維持)を満たす最大の学習強度を「動作点」とする。

- **セル A(L9 × lr=1e-5 × 1ep)** = **AVG 0.465(-0.004)** が最高。base 0.469 とは
  実質誤差(100 件 × 8 タスクで 1 タスク 1 件の変動 ≈ ±0.001×8=±0.008 相当)の範囲内で
  **事実上「base 維持」を達成**。
- 「厳密に AVG ≥ 0.469」を満たすセルは無いが、**base に 0.005 以内で並び、かつ
  ichikara の SFT を 1 epoch 通したセルが存在する**という事実が本 sweep の中心的所見。
- lr を上げるほど / 可変層を増やすほど AVG は base から遠ざかる単調傾向: L9 lr={1e-5,
  3e-5, 1e-4}=0.465/0.425/0.419、L6-9 lr={同}=0.443/0.441/0.364。sft-003 の
  1e-4×2ep 順位(L9 0.396、[6..9] 0.324)と比べ、**同じ lr でも epoch 半減で +0.02〜+0.10**
  の改善 = 学習量(lr × epoch × 可変 params)がドライバであることが定量確認された。

## 考察

### 1. 学習強度過剰仮説の直接検証

sft-003 と sft-004 は **layers・data・precision・seq_len・batch まで bit-identical、
差分は lr × epoch のみ**。この直接比較で:
- L9: 1e-4×2ep 0.396 → 1e-5×1ep **0.465(+0.069)**
- L6-9: 1e-4×2ep 0.324 → 1e-5×1ep **0.443(+0.119)**

sft-003 の下流劣化は、少なくとも本 few-shot 評価枠組みでは
**学習強度で説明できる比率が大きい**。lr を 10 倍下げ、epoch を半減
(実効的な累積学習量 1/20)で、base 差が -0.145 → -0.026 まで縮んだ。

### 2. lr × 層数の交互作用

- L9 の lr 感度: 0.465 → 0.425 → 0.419(1e-5 → 3e-5 → 1e-4)
- L6-9 の lr 感度: 0.443 → 0.441 → 0.364(1e-5 → 3e-5 → 1e-4)

**L6-9 は 3e-5 で「頑張ってもほぼ 1e-5 と同じ」で、1e-4 で急に崩れる**。単層 L9 の
場合は 1e-5 → 3e-5 の時点で既に -0.04 動く。可変 params が多い方が 3e-5 の消化
余力があり、逆に 1e-4 では 4 倍の勾配が乗って一気に劣化する非対称性が見える。

### 3. 検証A(生成劣化 3 段階)の観点

sft-003 で観測された劣化モード(様式ドリフト → エコー → 事実崩壊)は、
本 sweep の全 6 セルでは ool レベルに現れなかった(全タスク ool ≤ 22%)。
とくに sft-003 [6..9] で jsts pearson=0(数値抽出不能)だった現象が、本 sweep
[6..9] lr=1e-4 でも 0.452 と回復している。ichikara の様式適合が epoch=1 では
まだ支配的にならないことを示唆する。

### 4. 「SFT で何が得られたか」の未解決問題

L9 lr=1e-5 は base をほぼ完全に維持したが、**逆に言えば SFT で何が改善したかは
本評価では見えない**。ichikara 指示追従の獲得は MT-Bench-ja / J-MIFEval のような
チャット形式評価で測る必要があり、few-shot NLU タスクでは検知できない。この
「動作点は見つけたが SFT の便益が本 few-shot 評価では未検証」というのが sft-004
の最大の残課題。

## 次のステップ

以下は **本セッションの実測をもとに Fable5 エージェント(次回以降の判断委任先)へ
持ち込む争点**。動作点候補が見つかったため、issue で開くべき方向性は複数ある。

1. **A. 動作点 L9 lr1e-5 を Phase 4 DPO に投入し、指示追従は DPO で作り込む**
   — sft-004 を「few-shot 能力を毀損しない SFT の上限線を確定」の位置付けで完了扱いに
2. **B. 動作点で MT-Bench-ja / J-MIFEval 相当のチャット評価**を回し、
   SFT で本当に指示追従が付いたかを検証してから Phase 4 判断
3. **C. sft-002(#34、データ多様化)を並行実施し、様式過剰適合を根本回避**
4. **D. さらに低い lr(1e-6)や部分 epoch(0.5)を試して真の動作点最適化**

判断は Fable5 に委ねる(ユーザー方針: 完全な新規性が求められる場合は fable5 に相談)。
