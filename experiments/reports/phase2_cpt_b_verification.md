# Phase 2 cpt-B チェックポイント検証 (2026-07-10)

## ラン概要(cpt-1.2b-layerft-full-010)

- LFM2.5-1.2B-Base + 中央層 [7, 8] フル FT(arXiv:2607.01232 §4.3、可変 10.93%)
- データ: wikipedia ja+en packed 51,763 系列(~318M トークン)、seq 6144、lr 1e-4
- **9,864 / 12,941 steps(76.2%、~245M トークン)でユーザー判断により停止**(発散なし・十分学習と判断)
- 学習中の train loss: 2.39(開始)→ 2.04〜2.11(停止時)。grad_norm 0.55〜0.7 で全区間安定、NaN/スパイクなし

## チェックポイント検証

方法: 各チェックポイントを bf16 ロードし、(a) 学習データ外の日本語 4 文(京都・観光・祇園祭)の perplexity、(b) 「日本の四国地方には」の greedy 生成(50 トークン)を確認。

| checkpoint | ja_ppl | 生成品質の要点 |
|---|---|---|
| base(CPT なし) | 8.29 | 「四国島嶼の四国諸島と四国本土に分かれる」— 事実誤り + 循環 |
| ckpt-1000 | 8.99 | 「四国四国地震」— 幻覚。学習初期の一時的悪化 |
| ckpt-2000 | 8.08 | 島嶼の反復(末尾の � は 50 トークン打ち切りによるバイト断片、破損ではない) |
| ckpt-3000 | 8.55 | 「〜を参照」の反復 |
| ckpt-4000 | 8.40 | 列挙の崩れ(四国国/四国県/四国市…) |
| ckpt-5000 | 8.66 | 同様の列挙癖 |
| ckpt-6000 | 8.54 | 反復あり |
| ckpt-7000 | 8.39 | ほぼ整った列挙 |
| **ckpt-8000** | 8.23 | **「四国 - 愛媛県、徳島県、香川県、高知県」— 正しい 4 県を列挙(base は不可)** |
| **ckpt-9000** | **8.07** | 整った文(軽い反復 1 回)。ppl 最良 |

## 解釈

1. **発散なし・単調な健全性**: ppl は ckpt-1000 で一時悪化(8.99)した後、揺らぎつつ低下し ckpt-9000 で最良(8.07、base 比 -2.7%)。学習中の train loss とも整合
2. **事実知識の改善が観測できる**: base が答えられなかった四国 4 県を ckpt-8000 以降が正答。Wikipedia CPT の狙いどおりの効果
3. **改善幅は控えめ**: 元々日本語が強い Base に対し wiki 中心 ~245M トークンでの上積みは ppl -3% 程度。これは想定内で、cpt-B の価値判定は llm-jp-eval(JMMLU 等)での cpt-A/cpt-C 比較(Phase 2 ゲート)で行う
4. **採用候補: ckpt-9000**(ppl 最良・生成良好)。次点 ckpt-8000(事実性の見えが最良)

## 次のステップ(Phase 2 ゲート)

- ckpt-9000 を cpt-B 代表として llm-jp-eval(ベースライン評価と同一凍結条件: 8 タスク × 100 件 × 4-shot)で評価し、cpt-A(= Base 無改造 → SFT 前提)/ cpt-C(公式 JP)と比較
- 中間チェックポイントの ppl 計測は 4 文プローブなので参考値。ゲート判定は llm-jp-eval で行うこと
